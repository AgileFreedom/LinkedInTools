## Tools

### `linkedin-optimiser_v1.html`

A single-file, client-side LinkedIn Profile Optimiser tool. No backend required.

**What it does:**

- Takes a user's LinkedIn profile URL and three configuration inputs (target market, consulting niche, engagement type)
- Sends the data to the Claude API (Anthropic), which analyses the profile and generates a full positioning report
- Returns a scored report (0–100) including:
  - A critique of the current headline with identified weaknesses
  - An AI-generated optimised headline (value-first, outcome-led, calibrated to the target market)
  - A critique of the current About section
  - A fully rewritten About section (opens with client pain, names buyer types, strong CTA)
  - Three positioning insights: Biggest Asset, Biggest Gap, Quick Win
- Fires a silent email notification via [EmailJS](https://www.emailjs.com/) to the tool owner with the LinkedIn URL and full report (lead capture)
- Presents a CTA at the end of the report inviting users to book a discovery call to explore building their own autonomous AI lead gen engine

**Tech stack:**

- Pure HTML, CSS, JavaScript — no framework, no build step
- [Anthropic Claude API](https://docs.anthropic.com/en/api/getting-started) (`claude-sonnet-4-20250514`)
- [EmailJS Browser SDK v4](https://www.emailjs.com/docs/sdk/installation/) for client-side email notifications
- [Google Fonts](https://fonts.google.com/) (Epilogue + Lora)

**Key features:**

- Fully self-contained single `.html` file
- Dark, professional UI with responsive layout
- T&Cs modal (Agile Freedom Consulting Ltd, company no. 15529145)
- Copy-to-clipboard on all generated outputs
- Score bar with animated fill
- No server, no database, no login required

## Setup

### 1. Anthropic API

This tool calls the Anthropic API directly from the browser. To enable this:

- Sign up at [console.anthropic.com](https://console.anthropic.com)
- Create an API key
- The API key is handled by the Claude.ai artifact environment in development; for production deployment you will need a backend proxy to avoid exposing your key client-side (see [Security](#security) below)

### 2. EmailJS (lead capture notifications)

The tool sends a silent email to `info@agilefreedom.co.uk` each time a report is generated. To activate this:

1. Sign up at [emailjs.com](https://www.emailjs.com/) (free tier: 200 emails/month)
2. Add an email service and connect your inbox — note the **Service ID**
3. Copy your **Public Key** from Account settings
4. Create an email template using the variables below — note the **Template ID**
5. Open `linkedin-optimiser_v1.html` and replace the three placeholder values near the bottom of the `<script>` block:

```js
const EMAILJS_PUBLIC_KEY  = 'YOUR_EMAILJS_PUBLIC_KEY';
const EMAILJS_SERVICE_ID  = 'YOUR_EMAILJS_SERVICE_ID';
const EMAILJS_TEMPLATE_ID = 'YOUR_EMAILJS_TEMPLATE_ID';
```

**EmailJS template variables:**

| Variable | Description |
|---|---|
| `{{linkedin_url}}` | The URL entered by the user |
| `{{person_name}}` | Name extracted from the profile by Claude |
| `{{target_market}}` | Selected target market |
| `{{niche}}` | Selected consulting niche |
| `{{engagement}}` | Selected engagement type |
| `{{profile_score}}` | Score out of 100 |
| `{{score_label}}` | Positioning label (e.g. "Practitioner Not Yet Consultant") |
| `{{new_headline}}` | Optimised headline generated by Claude |
| `{{new_about}}` | Rewritten About section |
| `{{biggest_asset}}` | Asset insight |
| `{{biggest_gap}}` | Gap insight |
| `{{quick_win}}` | Quick win recommendation |
| `{{timestamp}}` | Date/time (Europe/London) |

## Security

> **Important:** This tool currently calls the Anthropic API directly from the browser. This is suitable for controlled demo environments, internal use, or platforms where the API key is injected at runtime (e.g. Claude.ai artifact environment).

**For public production deployment**, do not expose your Anthropic API key client-side. Instead:

- Create a lightweight backend proxy (e.g. a Vercel Edge Function, Cloudflare Worker, or simple Node.js/Express endpoint)
- The frontend sends the prompt to your proxy
- The proxy attaches the API key server-side and forwards to Anthropic
- This keeps your key secure and allows rate limiting and logging
